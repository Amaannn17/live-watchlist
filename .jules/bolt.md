## 2024-03-24 - Avoid innerHTML equality checks for DOM caching
**Learning:** Comparing `card.innerHTML === newInnerHTML` is an unreliable and inefficient way to prevent redundant DOM updates during polling. Browsers normalize HTML when setting `innerHTML` (e.g., reordering attributes, converting quotes, standardizing tags). Thus, the read-back `innerHTML` string often won't strictly equal the raw template string you inserted, causing false positive comparisons and triggering unnecessary DOM repaints on every poll cycle.
**Action:** When caching HTML strings to prevent unnecessary DOM updates, store the raw generated string in a custom property on the element (e.g., `card._rawHtml`) instead of comparing against the browser-normalized `innerHTML` or bloating DOM nodes with massive data-* attributes.

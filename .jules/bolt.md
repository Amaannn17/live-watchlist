## 2026-07-21 - [Unreliable innerHTML checks causing unnecessary re-renders]
**Learning:** Comparing `card.innerHTML !== newInnerHTML` is unreliable because browsers can serialize `innerHTML` differently (e.g., adding closing tags, changing attribute quoting, or altering spacing) than the raw HTML string assigned to it. This unreliability causes unnecessary DOM updates during every polling cycle, even when the data hasn't actually changed, hurting performance.
**Action:** Instead of relying on `innerHTML` equality checks, cache the generated raw HTML string in a custom property on the element (e.g., `card._rawHtml`) and compare against that cached string. This avoids browser serialization discrepancies and prevents unnecessary re-renders, while keeping DOM nodes free of massive dataset attributes.
